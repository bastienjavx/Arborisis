# Plan — Web Radio Arborisis

## Objectif
Créer une page `/radio` avec un vrai flux audio continu écoutable depuis n'importe quel lecteur (VLC, iTunes, navigateurs, mobiles).

## Architecture

### 1. Backend — Laravel

#### a. Routes (`routes/web.php`)
- `GET /radio` → `RadioController@index` (page Inertia)
- `GET /radio/stream` → `RadioController@stream` (flux audio continu MP3/OGG)
- `GET /radio/stream.m3u` → `RadioController@playlist` (fichier M3U pour VLC)
- `GET /api/radio/now-playing` → `RadioController@nowPlaying` (JSON métadonnées)

#### b. `RadioController`
- `index()` : retourne `Inertia::render('Radio/Index')` avec les métadonnées initiales
- `stream()` : ouvre un flux continu HTTP avec headers ICY, boucle infinie sur la playlist
- `playlist()` : génère un fichier `.m3u` avec l'URL du stream
- `nowPlaying()` : retourne JSON { title, artist, cover, duration, started_at }

#### c. `RadioStreamService` (nouveau, `app/Services/Radio/`)
Responsabilités :
- Construire la playlist depuis `Sound::public()` (status=published, visibility=public)
- Gérer la rotation aléatoire / round-robin
- Tracker le "now playing" dans Redis/Cache avec TTL
- Générer les URLs audio temporaires S3 à la volée
- Lire et streamer les fichiers audio chunk par chunk
- Injecter les métadonnées ICY (titre, artiste) toutes les N secondes

Méthodes clés :
- `getPlaylist(): Collection<Sound>`
- `getNextSound(): Sound`
- `getCurrentSound(): ?Sound` (depuis Redis)
- `streamToOutput($outputStream): void` — boucle de lecture S3 → client
- `generateIcyMetadata(string $title, string $artist): string`

#### d. Modèle / Cache
- Clé Redis `radio:now-playing` → JSON { sound_id, started_at, url_expires_at }
- Clé Redis `radio:listeners` → compteur d'auditeurs (inc/déc sur connexion)
- Clé Redis `radio:history` → liste des N derniers sons joués

#### e. Configuration
Ajouter dans `config/radio.php` :
- `icy_metaint` (8192 par défaut)
- `chunk_size` (8192)
- `crossfade_enabled` (false pour V1)
- `fallback_sounds` (IDs de sons de secours)

### 2. Frontend — Vue 3

#### a. Page `resources/js/Pages/Radio/Index.vue`
Layout `GuestLayout`, contenu :
- **Visualiseur radio** : cover du son en cours, titre, créateur
- **Player natif HTML5** connecté à `/radio/stream` (pas le Pinia store, car c'est un flux continu)
- **Boutons** : play/pause, volume, mute
- **Info en direct** : son en cours, prochain son, historique (5 derniers)
- **Compteur d'auditeurs** (polling sur `/api/radio/now-playing`)
- **Lien partage** : copier l'URL du stream ou télécharger le M3U

#### b. Store Pinia dédié (optionnel)
`useRadioStore` — si besoin de state local pour l'UI (volume, mute, historique)

### 3. Flux technique — `GET /radio/stream`

1. Headers de réponse :
   ```
   Content-Type: audio/mpeg
   Cache-Control: no-cache, no-store
   Connection: close
   icy-name: Arborisis Radio
   icy-genre: Nature / Field Recording
   icy-metaint: 8192
   ```
2. `set_time_limit(0)` + `ignore_user_abort(true)`
3. Boucle :
   - Récupérer le son suivant via `RadioStreamService`
   - Générer URL S3 temporaire (60 min)
   - Ouvrir le fichier avec `fopen($url, 'r')`
   - Streamer par chunks de 8KB
   - Tous les 8192 octets, injecter le bloc ICY metadata
   - Quand le fichier finit, passer au suivant sans couper la connexion
4. Gestion des erreurs :
   - Si S3 indisponible → son fallback
   - Si aucun son → stream silence (ou message audio)
   - Si client déconnecte (`connection_aborted()`) → sortir proprement

### 4. Fichier M3U (`/radio/stream.m3u`)
```
#EXTM3U
#EXTINF:-1,Arborisis Radio
https://arborisis.com/radio/stream
```

## Questions ouvertes
- **Timeout Nginx/PHP-FPM** : en production, il faudra vérifier que le proxy/web server accepte les connexions longues. Si ce n'est pas possible, on peut fallback sur une approche HLS (segments) ou sur un simple endpoint qui redirige vers le fichier en cours.
- **Format audio** : les fichiers sont-ils tous en MP3 ? Si formats variés (WAV, FLAC, OGG), il faut soit les convertir, soit streamer tel quel avec un header `audio/mpeg` générique (risque de compatibilité). À vérifier.
- **Crossfade** : V1 sans crossfade (coupe net entre les sons). V2 possible avec FFmpeg.
- **Pub/Sub metadata** : pour le compteur d'auditeurs et les metadata temps réel, utiliser polling toutes les 5s ou SSE (Server-Sent Events). SSE est préférable.
