# Plan: Correction du dashboard admin Filament

## Problèmes identifiés

### 1. Erreurs d'enums castés dans les colonnes de table (CRITIQUE)
Quand un modèle Laravel cast un attribut en `BackedEnum`, Filament v3 passe l'objet enum (pas la string) aux callbacks `formatStateUsing` et `color`. Or toutes les ressources appellent `Enum::from($state)` sur une valeur qui est **déjà** l'enum, ce qui lève une `TypeError`.

**Fichiers concernés:**
- `app/Filament/Resources/UserResource.php` — colonne `role`
- `app/Filament/Resources/SoundResource.php` — colonnes `status`, `visibility`
- `app/Filament/Resources/ReportResource.php` — colonnes `reason`, `status`
- `app/Filament/Resources/EchoTransactionResource.php` — colonnes `type`, `status`

**Correction:** remplacer `fn (string $state)` par `fn (UserRole $state)` (ou l'enum correspondant) et utiliser directement `$state->label()` / `$state` dans le `match`.

### 2. Méthodes de table appelées sur des champs de formulaire (CRITIQUE)
`TextInput` n'a pas de méthode `sortable()`.

**Fichiers concernés:**
- `app/Filament/Resources/CategoryResource.php` (form, ligne ~60)
- `app/Filament/Resources/EnvironmentResource.php` (form, ligne ~60)

**Correction:** supprimer les appels `->sortable()` sur les `TextInput`.

### 3. Champ formulaire invalide dans ReportResource
`TextInput::make('resolver.name')` n'est pas un attribut du modèle `Report`. Filament va essayer d'écrire dans `resolver.name` lors de la sauvegarde et va planter.

**Fichier:** `app/Filament/Resources/ReportResource.php` (form)

**Correction:** remplacer par un `Placeholder` ou un champ `TextInput` sur la relation via `->afterStateHydrated()` et `->dehydrated(false)`.

### 4. Routes `create` manquantes pour Wallet & EchoTransaction
Les pages `ListWallets` et `ListEchoTransactions` incluent un `CreateAction::make()`, mais `getPages()` de leur resource ne déclare pas de route `create`. Le clic sur "Créer" mène vers une 404.

**Fichiers:**
- `app/Filament/Resources/WalletResource.php`
- `app/Filament/Resources/EchoTransactionResource.php`

**Correction:** soit ajouter `'create'` dans `getPages()`, soit retirer le `CreateAction::make()` des pages list.

### 5. Couleur `secondary` inexistante dans Filament 3
`EchoTransactionResource::table()` utilise `'secondary'` comme couleur de badge. Filament 3 ne connaît pas cette couleur (c'est `gray`).

**Fichier:** `app/Filament/Resources/EchoTransactionResource.php`

**Correction:** remplacer `'secondary'` par `'gray'`.

### 6. Potentiel crash sur `WalletResource::table()` — colonne balance
Le cast `decimal:2` du modèle peut parfois passer un `float` à la closure au lieu d'une `string`.

**Fichier:** `app/Filament/Resources/WalletResource.php`

**Correction:** élargir le type-hint en `string|float|int`.

### 7. Cache / bootstrap potentiellement corrompu
Les logs historiques montrent des erreurs de boot Filament (`Call to a member function getPage() on string`, `accessDeniedRedirectUrl`). Même si le code source semble corrigé maintenant, il faut vider les caches Laravel pour s'assurer qu'aucun vieux cache ne pollue l'état.

## Étapes d'exécution

1. **Corriger les enums castés** dans les 4 ressources concernées.
2. **Corriger les appels `sortable()`** dans `CategoryResource` et `EnvironmentResource`.
3. **Corriger le champ `resolver.name`** dans `ReportResource`.
4. **Corriger les routes create** dans `WalletResource` et `EchoTransactionResource`.
5. **Corriger la couleur `secondary`** dans `EchoTransactionResource`.
6. **Corriger le type-hint balance** dans `WalletResource`.
7. **Vider les caches** Laravel (`config`, `route`, `view`, `filament`).
8. **Vérifier** en accédant à `/admin` et naviguer entre les ressources.
