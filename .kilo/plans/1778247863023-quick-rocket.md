# Plan : Module Data Science Audio — Arborisis

## Résumé
Implémentation d'un module complet de data science orienté audio avec backend Python (librosa, scipy, matplotlib, seaborn, pandas, numpy), intégration Laravel via jobs/processes, et tableau de bord Vue 3 avec analyse temps réel Web Audio API. Architecture hybride : pré-calcul à l'upload + recalcul manuel avec paramètres configurables. Accessible aux créateurs (dashboard privé complet) et aux visiteurs (aperçu public limité).

## Phase 1 : Fondations — Backend Python & Dépendances

### 1.1 Créer l'environnement Python
- Créer `<redacted>/python/requirements.txt` avec : `librosa`, `scipy`, `numpy`, `matplotlib`, `seaborn`, `pandas`, `soundfile`, `Pillow`
- Documenter l'installation dans un README (`<redacted>/python/README.md`)
- Créer `<redacted>/python/config.py` pour les chemins, paramètres par défaut (sample rate, n_fft, hop_length, n_mels, n_mfcc)

### 1.2 Module `core/`
- **`loader.py`** : Fonction `load_audio(path, sr=None, mono=True)` supportant WAV/MP3/FLAC via librosa/soundfile. Gestion des formats via mimetype. Pour les flux temps réel, accepter des bytes/array numpy.
- **`preprocessing.py`** :
  - `apply_filter(y, sr, filter_type, cutoff)` — passe-bas, passe-haut, passe-bande (scipy.signal)
  - `normalize(y, method='peak'|'rms'|'zscore')`
  - `detect_voice_activity(y, sr, frame_length, hop_length, energy_threshold)` — VAD basé énergie + option zcr
  - `trim_silence(y, sr, top_db=20)` — wrapper librosa.effects.trim
- **`segmentation.py`** :
  - `segment_frames(y, sr, frame_length, hop_length)` — sliding window
  - `segment_onsets(y, sr, hop_length)` — détection onset librosa
- **`utils.py`** :
  - `save_figure(fig, output_path, dpi=150, format='png')`
  - `load_json(path)`, `save_json(data, path)`
  - `ensure_dir(path)`

### 1.3 Module `features/`
- **`temporal.py`** :
  - `extract_zcr(y, frame_length, hop_length)` — zero-crossing rate
  - `extract_rms(y, frame_length, hop_length)` — root mean square energy
  - `extract_envelope(y, sr, frame_length)` — enveloppe d'amplitude
  - `extract_temporal_features(y, sr)` — agrège toutes les features temporelles
- **`spectral.py`** :
  - `extract_spectral_centroid(y, sr, n_fft, hop_length)`
  - `extract_spectral_bandwidth(y, sr, n_fft, hop_length)`
  - `extract_spectral_rolloff(y, sr, n_fft, hop_length, roll_percent=0.85)`
  - `extract_spectral_contrast(y, sr, n_fft, hop_length)`
  - `extract_spectral_flatness(y, n_fft, hop_length)`
  - `extract_spectral_features(y, sr, n_fft, hop_length)` — agrège tout
- **`cepstral.py`** :
  - `extract_mfcc(y, sr, n_mfcc, n_fft, hop_length, n_mels)`
  - `extract_mfcc_delta(mfccs, order=1|2)` — delta et delta-delta
  - `extract_cepstral_features(y, sr, n_mfcc, n_fft, hop_length)` — agrège tout

### 1.4 Module `visualizations/`
- **`spectrograms.py`** :
  - `plot_stft(y, sr, n_fft, hop_length, scale='linear'|'log', output_path)`
  - `plot_mel_spectrogram(y, sr, n_fft, hop_length, n_mels, output_path)`
  - `plot_mfcc(mfccs, output_path)`
  - `plot_combined_spectrograms(y, sr, config, output_dir)` — génère les 3
- **`heatmaps.py`** :
  - `plot_feature_correlation(features_df, output_path)` — heatmap corrélation seaborn
  - `plot_confusion_matrix(cm, labels, output_path, normalize=True)` — matrice confusion
- **`activation_maps.py`** :
  - `plot_activation_placeholder(input_shape, output_path)` — placeholder pour futures cartes d'activation DL

### 1.5 Pipeline end-to-end
- **`pipeline.py`** : Fonction `run_analysis(audio_path, output_dir, config_dict)` orchestrant :
  1. Chargement audio
  2. Prétraitement (filtrage, normalisation, VAD optionnel)
  3. Segmentation
  4. Extraction features (temporelles + spectrales + cepstrales)
  5. Génération visualisations (spectrogrammes + heatmaps)
  6. Export données JSON + images PNG
- **`cli.py`** : Point d'entrée CLI acceptant `--input`, `--output`, `--config` (JSON string) et retournant un JSON avec chemins et features

## Phase 2 : Backend Laravel — Modèles, Migrations & Services

### 2.1 Migrations
- **`create_sound_analyses_table`** :
  - `id`, `sound_id` (FK, unique, cascade)
  - `status` (AnalysisStatus enum)
  - `features_json` (JSONB) — stockage des features numériques agrégées
  - `features_detailed_json` (JSONB) — séries temporelles (optionnel, limité)
  - `parameters_json` (JSONB) — paramètres utilisés pour l'analyse
  - `processed_at`, `failed_reason` (nullable text)
  - `timestamps()`
- **`create_sound_visualizations_table`** :
  - `id`, `sound_analysis_id` (FK, cascade)
  - `type` (enum: STFT, MEL_SPECTROGRAM, MFCC, FEATURE_CORRELATION, CONFUSION_MATRIX)
  - `scale` (enum: LINEAR, LOG, MEL) — pour spectrogrammes
  - `file_path`, `disk`
  - `parameters_json` (JSONB)
  - `timestamps()`

### 2.2 Enums PHP
- **`SpectrogramType`** : STFT, MEL_SPECTROGRAM, MFCC
- **`FrequencyScale`** : LINEAR, LOG, MEL
- **`AnalysisStatus`** : PENDING, COMPLETED, FAILED

### 2.3 Modèles Eloquent
- **`SoundAnalysis`** : belongsTo Sound, hasMany SoundVisualization
- **`SoundVisualization`** : belongsTo SoundAnalysis

### 2.4 Services
- **`PythonRunnerService`** :
  - Méthode `run(string $script, array $args): array` utilisant `Symfony\Component\Process\Process` ou `shell_exec`
  - Gestion timeout, stderr, parsing JSON stdout
  - Validation que Python et les dépendances sont installées
- **`AudioAnalysisService`** :
  - `analyze(Sound $sound, array $config = []): SoundAnalysis` — orchestre l'analyse complète
  - `getOrCreateAnalysis(Sound $sound): SoundAnalysis` — retourne existante ou crée
  - `generateVisualization(SoundAnalysis $analysis, string $type, array $params): SoundVisualization`
  - `exportFeatures(SoundAnalysis $analysis, string $format = 'json'|'csv')`
- **`FeatureExtractorService`** :
  - `getTemporalFeatures(SoundAnalysis $analysis): array`
  - `getSpectralFeatures(SoundAnalysis $analysis): array`
  - `getCepstralFeatures(SoundAnalysis $analysis): array`
  - `getAllFeatures(SoundAnalysis $analysis): array`

### 2.5 Job & Event
- **`ProcessAudioAnalysis` (implements ShouldQueue)** :
  - Reçoit `sound_id` + `config`
  - Télécharge le fichier audio temporairement (S3 → local)
  - Exécute le pipeline Python
  - Parse le JSON de sortie
  - Crée `SoundAnalysis` + `SoundVisualization`
  - Nettoie les fichiers temporaires
  - Gère les erreurs (mark FAILED avec raison)

### 2.6 Intégration upload existant
- Modifier `SoundUploadService::upload()` pour dispatcher `ProcessAudioAnalysis` après création du `SoundFile`
- Ajouter un champ optionnel `analysis_on_upload` (booléen, default true)

## Phase 3 : API & Routes Laravel

### 3.1 Form Request
- **`AnalyzeSoundRequest`** :
  - `sound_id` (required, exists)
  - `window_size` / `n_fft` (int, power of 2, default 2048)
  - `hop_length` (int, default 512)
  - `frequency_scale` (enum: linear, log, mel)
  - `n_mels` (int, default 128)
  - `n_mfcc` (int, default 13)
  - `preprocessing` (array optionnel: filter_type, cutoff, normalize, vad)
  - `visualization_types` (array of enum)

### 3.2 Controller
- **`AudioAnalysisController`** :
  - `show(Sound $sound)` — retourne l'analyse existante + visualisations (Inertia pour dashboard, JSON pour API)
  - `analyze(Sound $sound, AnalyzeSoundRequest $request)` — déclenche analyse manuelle avec paramètres
  - `export(SoundAnalysis $analysis, string $format)` — export JSON/CSV/PNG
  - `realtimeData(Sound $sound)` — retourne les données brutes nécessaires pour l'analyse temps réel frontend (URL audio, durée, sr si connue)

### 3.3 Routes
- `GET /sounds/{sound}/analysis` → `AudioAnalysisController@show` (public, visiteurs voient un aperçu)
- `POST /sounds/{sound}/analysis` → `AudioAnalysisController@analyze` (auth + policy)
- `GET /sounds/{sound}/analysis/export` → `AudioAnalysisController@export` (auth + policy)
- `GET /api/sounds/{sound}/analysis` → JSON API publique (aperçu limité)

### 3.4 Policy
- **`SoundAnalysisPolicy`** :
  - `view(User $user, Sound $sound)` : public si son public, ou si owner/admin
  - `analyze(User $user, Sound $sound)` : owner ou admin uniquement
  - `export(User $user, SoundAnalysis $analysis)` : owner ou admin

## Points d'attention / Risques

1. **Performance Python** : L'analyse d'un fichier audio de plusieurs minutes peut prendre 5-30s. Obligatoire de passer par une queue (Laravel Queue + Redis) pour ne pas bloquer les requêtes HTTP.
2. **Stockage visualisations** : Les images PNG générées peuvent être volumineuses. Stocker sur le même disk que les audio (S3 'audio') avec nettoyage automatique si l'analyse est recalculée.
3. **Compatibilité formats** : librosa utilise ffmpeg pour MP3. S'assurer que ffmpeg est installé sur le serveur de production (Contabo).
4. **Sécurité** : Ne jamais passer des chemins utilisateur directement au script Python. Toujours utiliser les paths Laravel/Storage et valider.
5. **Web Audio API** : Le décodage de MP3/FLAC dans le navigateur nécessite le CORS correct sur les URLs S3. Vérifier les headers CORS du bucket Contabo.
6. **Memory** : Les arrays numpy pour de longs fichiers audio peuvent consommer beaucoup de RAM. Implémenter une limite de durée (ex: 10 min max pour analyse complète) ou du streaming/chunking.
